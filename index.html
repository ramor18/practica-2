Etiquetas HTML

By: Ramona Mordan

  1. !DOCTYPE html
    2. Indica al navegador que el documento está basado en el estándar HTML5.
  2. html /html
    3. Representa la raíz de un documento HTML. Todos los demás elementos de la estructura HTML deben ser recogidos dentro de estas etiquetas.
  3. p /p
    4. Etiqueta usada para escribir párrafos de texto.
  4. body /body
    5. Al contrario que la etiqueta de metadatos head, todo lo que quieras mostrar en la página web debe ir recogido dentro de las etiquetas de apertura y cierre de body. Este contenido será el que se muestre en la web.
  5. head /head
    6. Representa una colección de metadatos acerca del documento, incluyendo enlaces a, o definiciones de, scripts y hojas de estilo. El resto de etiquetas de metadatos, irán recogidas dentro de las etiquetas de apertura y cierre del head. Importante explicar que estos metadatos del documento, es información para el navegador y no contenido que será visible en la página web. A excepción de la etiqueta title que veremos a continuación.
  6. title /title
    7. Etiqueta usada para definir el título de la página web.

  7. ol /ol
    8. Etiquetas para crear una lista ordenada
  8. ul /ul
    9. Etiquetas para crear una lista des-ordenada
  9. li /li
    10. Etiquetas que recogen el contenido de un elemento de una lista, sea ordenada o no.

    Ejemplos de estas etiquetas:


  10. dl /dl
    11. Usada para crear una lista de definiciones.
  11. dt /dt
    12. Representa un término definido por la siguiente etiqueta dd
  12. dd /dd
    13. Se usa para definir los términos listados antes que él.

    Ejemplos de estas etiquetas:


  13. div /div
    14. Etiqueta común utilizada para crear un contenedor genérico.

    Ejemplo de esta etiqueta:


  14. h1,h2,h3,h4,h5,h6
    15. Son etiquetas HTML muy importantes, ya que son usadas para jerarquizar el contenido de la web. Las etiquetas se usan para explicar brevemente el contenido que irá a continuación.

    Ejemplo de esta etiqueta:


  15. nav /nav
    16. Usadas para definir el contenido que será la sección de navegación de la web.
  16. a /a
    17. Etiqueta utilizada para crear hiperenlaces en el documento HTML.

    Ejemplo de estas etiquetas:


  17. img
    18. Etiqueta para «pintar» una imagen en la página web.
  18. center /center
    19. Se utiliza para centro de alinear el texto.

    Ejemplo de estas etiquetas:


  19. u /u
    20. Usada para definir texto estilísticamente diferente del texto normal o nombres propios en otros idiomas.
  20. em
    21. Es apropiado para marcar con énfasis las partes importantes de un texto.

    Ejemplo de estas etiquetas:


  21. style /style
    22. Etiquetas usadas para introducir código CSS en línea, es decir, en el propio documento HTML.
  22. table /table
    23. Etiquetas de apertura y cierre de una tabla. El resto de etiquetas de la tabla han de ir siempre recogidas entre estas dos etiquetas.
  23. td /td
    24. Usada para definir una celda de una tabla.
  24. th /th
    25. Etiqueta que se usa para definir el encabezado de una celda.
  25. thead /thead
    26. Indica el bloque de filas que describen las etiquetas de las columnas de la tabla.

    Ejemplo de estas etiquetas:


  26. hr
    27. Etiqueta utilizada para «romper» entre dos secciones de una web. Usada comúnmente como separador.
  27. strong /strong
    28. Etiqueta para definir una palabra o conjunto de ellas como importantes. Tiene una fuerte importancia en el SEO de la página.
  28. sub /sub y sup /sup
    29. Etiquetas utilizadas para representar un subíndice o superíndice.
  29. big /big
    30. Muestra el texto marcado con un tamaño de fuente más grande.

    Ejemplo de estas etiquetas:


  30. link
    31. Se usa para enlazar recursos externos al documento HTML. El ejemplo más común son las hojas de estilos CSS.
  31. meta
    32. Etiqueta usada para definir otros metadatos que no se pueden definir con una etiqueta HTML especifica. Por ejemplo para definir el autor del sitio, o la descripción del mismo.
  32. script
    33. Es usado para insertar códigos de scripts del lado cliente en un documento. Esto puede ser logrado de dos formas: insertando el código directamente como contenido de este elemento o refiriéndose a un archivo externo a través del atributo "src", que contenga el código del script.

    Ejemplo de estas etiquetas:


  33. video /video
    34. Se usa para reproducir video en la página web junto a sus archivos de audio y capciones asociadas.
  34. source
    35. Permite a autores especificar recursos multimedia alternativos para las etiquetas de video o audio.
  35. track
    36. Se utiliza como elemento hijo de los elementos multimedia, audio y video. Le permite especificar pistas de texto cronometradas (o datos basados en el tiempo), por ejemplo, para manejar subtítulos automáticamente.

    Ejemplo de estas etiquetas:


  36. map
    37. Es una etiqueta de contenedor que indica el comienzo y el final de la descripción del mapa de imagen.

    Ejemplo de estas etiquetas:


  37. figure /figure
    38. Indica una figura ilustrada como parte del documento HTML5.
  38. figcaption /figcaption
    39. Utilizada para definir la leyenda de una figura.
  39. main /main
    40. Se usa para definir el contenido principal del documento. Solamente puede existir uno por documento.
  40. section /section
    41. Define una sección del documento
  41. article /article
    42. Define contenido independiente de la web.
  42. aside /aside
    43. Dentro de estas etiquetas suele alojarse el contenido adicional de la web. Suele ser contenido relacionado con la web pero de poca importancia
  43. header /header
    44. Se usan para definir la cabecera la página web. Suele contener el logotipo, menú de navegación, etc.
  44. footer /footer
    45. Usadas para definir el pie de página.
  45. small /small
    46. Utilizada para dejar un comentario aparte, del tipo una nota de derechos de autoría, u otros textos que no son esenciales para la comprensión del documento.
  46. cite /cite
    47. Para indicar el título de una obra
  47. mark /mark
    48. Usada para resaltar texto
  48. span /span
    49. Etiqueta HTML sin ningún significado específico. Se usa conjuntamente con los atributos «class» o «id» para atribuirle ciertas características.
  49. br
    50. Etiqueta utilizada para crear un salto de línea
  50. iframe /iframe
    51. Es una etiqueta que sirve para anidar otro documento HTML dentro del documento principal.
  51. object /object
    52. Utilizada llamar a un recurso externo de la web. Este recurso será tratado como una imagen, o un recurso externo para ser procesado por un plugin.
  52. audio /audio
    53. Usada para cargar en una web un archivo de audio o stream de audio.
  53. svg /svg
    54. Se usa para llamar a una imagen vectorizada.
  54. caption /caption
    55. Usada para indicar el título de la tabla.
  55. colgroup /colgroup
    56. Etiqueta utilizada para agrupar dos o más columnas de una tabla.
  56. tbody /tbody
    57. Usada para describir los datos concretos de una tabla.
  57. tfoot /tfoot
    58. Indica los bloques de filas que describen los resúmenes, o datos totales de una columna de una tabla.
  58. tr /tr
    59. Se usa para indicar una fila de celdas de una tabla.
  59. form /form
    60. Etiqueta de apertura y cierre de un formulario de una página web. El resto de etiquetas de formulario deben ir siempre recogidas entre estas etiquetas de apertura y cierre de formulario.
  60. fieldset /fieldset
    61. Etiqueta que representa un conjunto o agrupación de elementos de un formulario. «Pinta» un recuadro alrededor de las etiquetas que estén contenidas dentro del fieldset
  61. legend /legend
    62. Etiqueta ligada a fieldset. Indica el título del fieldset
  62. label /label
    63. Se usa para definir el nombre o título de un control del formulario.
  63. input
    64. Pinta un campo de introducción de datos para el usuario. Es de las principales etiquetas de un formulario.
  64. button /button
    65. Etiqueta utilizada para representar un botón en el formulario.
  65. select /select
    66. Input que permite una selección entre un conjunto de opciones.
  66. option /option
    67. Etiqueta ligada a select. Permite añadir diferentes opciones al select
  67. textarea /textarea
    68. Añade un campo al usuario para que pueda introducir texto en unas líneas máximas de texto que el desarrollador puede definir.
  68. abbr
    69. (o Elemento de Abreviación HTML) representa una abreviación o acrónimo.
  69. !--...--
    70. Utilizado para agregar comentarios.
  70. bdi
    71. Es la abreviatura de aislamiento bidireccional, aísla una parte del texto que puede formatearse en una dirección
  71. bdo
    72. Permite a los autores anular el algoritmo bidireccional para fragmentos específicos de texto.
  72. code
    73. Inserta texto que representa código de computadora. Puede ser útil para mostrar funciones, código de programación o variables.
  73. del
    74. Marca las partes de un texto o documento que han sido suprimidas o sustituidas.
  74. dfn
    75. Sirve para marcar el término que se quiere definir.
  75. i
    76. Es un espacio de texto que representa un cambio en el estado de ánimo o la calidad del texto.
  76. ins
    77. Marca las partes de un texto que han sido añadidos al documento. Sus etiquetas son: ins e /ins (ambas obligatorias).
  77. kbd
    78. Contiene texto que corresponde a una entrada del teclado.
  78. mark
    79. Representa un texto marcado o resaltado como referencia o anotación, debido a su relevancia o importancia en un contexto particular.
  79. meter
    80. Indicar una medida dentro de un rango; es decir debe ser acotada, tener un inicio y un fin.
  80. pre
    81. Representa texto preformateado.
  81. progress
    82. Se utiliza para mostrar el progreso de una tarea, ya sea en forma de barra de progreso o de un porcentaje.
  82. q
    83. Indica que el texto adjunto es una cita corta en línea.
  83. s
    84. Muestra el texto tachado con una linea horizontal.
  84. samp
    85. Se utiliza para incluir texto en línea que representa una muestra (o cita) de la salida de un programa de ordenador.
  85. template
    86. Es un mecanismo para mantener el contenido HTML del lado del cliente que no se renderiza cuando se carga una página.
  86. time
    87. Representa un periodo específico en el tiempo.
  87. u
    88. Se utiliza para definir texto subrayado.
  88. var
    89. Contiene texto que supone ser la instancia de una variable o el argumento de un programa.
  89. wbr
    90. Representa una posición dentro del texto donde el explorador puede opcionalmente saltar una línea.
  90. data
    91. Vincula un contenido dado con una traducción legible por una máquina.
  91. datalist
    92. Contiene un conjunto de elementos option que representan los valores disponibles para otros controles.
  92. details
    93. Es usado como un widget de revelación a través del cual el usuario puede obtener información adicional .
  93. dialog
    94. Representa una caja de diálogo u otro componente interactivo, como inspector o ventana.
  94. dir
    95. Era usado para crear directorios en multicolumna.
  95. output
    96. Es usado para mostrar resultados de cálculos dentro de un formulario.
  96. optgroup
    97. Permite a los autores agrupar opciones (elemento HTML option) en una lista de opciones (elemento HTML select). Un título es usualmente mostrado encima de las opciones contenidas por este elemento.
  97. noscript
    98. Es usado para proveer contenido alternativo para ser mostrado cuando un script no es ejecutado.
  98. rp
    99. Esta etiqueta define lo que muestra si un navegador no soporte anotaciones ruby.
  99. rt
    100. Es parte del conjunto de etiquetas “ruby” (ruby annotation) que se utilizan para proporcionar anotaciones (generalmente en forma de símbolos fonéticos o glosas) para caracteres específicos en un texto.
  100. ruby
    101. Es una de las menos conocidas y utilizadas sobre todo en Occidente. Esto se debe a que se utiliza sobre todo en idiomas asiáticos. La utilidad de la etiqueta ruby es la de mostrar anotaciones sobre la pronunciación de un texto o palabra.
  101. s
    102. Representa contenido que ya no es relevante o preciso. Este elemento puede ser útil cuando el autor necesita hacer evidente un cambio en los precios, la disponibilidad, las políticas, etc. (Desestimado en HTML5)
  102. acronym
    103. Define a su contenido como un acrónimo, y es comúnmente utilizado junto con el atributo "title" para expandir/explicar al acrónimo.
